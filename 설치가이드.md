## 🔧 STEP 2 — Firestore 보안 규칙 설정

Firebase 콘솔 > Firestore Database > **규칙** 탭

`firestore.rules` 파일 내용을 붙여넣고 **게시**

> `/config/missions`, `/config/students` 문서를 직접 갱신합니다.
> 따라서 `config` 경로 쓰기 권한이 반드시 필요합니다.

## 🔧 STEP 3 — Apps Script 설정

### 3-1. 코드 설치

1. Google Sheets > **확장 프로그램** > **Apps Script**
2. `apps-script.js` 내용 전체 붙여넣기
3. 2번 줄 수정:
   ```js
   const FIREBASE_PROJECT_ID = "your-project-id"; // ← 변경
   ```

### 3-2. 웹앱으로 배포 (점수 역기록용)

1. 오른쪽 위 **배포** > **새 배포**
2. 유형: **웹앱** 선택
3. 설명: `활동 트래커 웹앱`
4. 다음 사용자로 실행: `나`
5. 액세스 권한: **모든 사용자 (익명 포함)**
6. **배포** 클릭
7. 생성된 URL 복사 → 앱 설정에 사용

### 3-3. 최초 동기화

1. `syncToFirebase` 함수 선택 > **실행**
2. 첫 실행 시 권한 허용 필요

## 🔧 STEP 4 — 웹앱 배포

### 방법 A: Firebase Hosting (권장)

```bash
npm install -g firebase-tools
firebase login
firebase init hosting
# public 폴더에 index.html, tracker.html, admin.html 복사
firebase deploy
```

→ `https://your-project.web.app` 생성

### 방법 B: GitHub Pages (무료)

1. GitHub 저장소 생성
2. 3개 HTML 파일 업로드
3. Settings > Pages > main 브랜치 선택

→ `https://username.github.io/repo` 생성

---

## 🔧 STEP 5 — 앱 최초 설정

1. 배포된 URL로 접속
2. 설정 입력:
   - **Firebase 프로젝트 ID**: Firebase 콘솔 > 프로젝트 설정
   - **Web API Key**: Firebase 콘솔 > 프로젝트 설정 > 웹 API 키
   - **Google Sheets ID**: 시트 URL의 `/d/` 다음 긴 문자열
   - **Apps Script URL**: STEP 3-2에서 복사한 URL
3. **시작하기** 클릭 → 이후 자동 저장됨

## ✨ 기능 요약

| 기능 | 설명 |
|------|------|
| 터치 완료 | 1번 터치 = 초록 완료, 다시 터치 = 취소 (토글) |

| Sheets 역기록 | 터치할 때마다 Apps Script로 점수 시트 자동 업데이트 |
| 실시간 동기화 | Firebase로 모든 기기에서 동시 반영 |
| 학생별 진행률 | 메인 화면에서 학생카드에 진행률 바 표시 |

## ❓ 자주 묻는 문제

**Q. 시트 동기화 오류**  
→ 시트가 **공개 링크**로 공유되어 있는지 확인  
→ 시트명이 정확히 `missions`, `students` 인지 확인

**Q. 점수가 시트에 안 써짐**  
→ Apps Script가 웹앱으로 배포되었는지 확인  
→ 배포 설정에서 "모든 사용자" 접근 권한 확인

**Q. 잠금 시간이 작동 안 함**  
→ B열 셀 서식: 일반 or 텍스트로 변경 후 재입력  
→ Apps Script 재동기화 실행

**Q. 관리자 PIN 분실**  
→ 브라우저 개발자 도구 > Application > LocalStorage > `admin_pin` 값 직접 수정
