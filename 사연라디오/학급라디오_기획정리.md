## 1. 기획 출발점

### 요청 사항 (원안)
- 사연 입력 페이지 / 재생 부스 페이지 **분리**
- 사연에 **내용 + 곡제목 + 가수** 입력, 순차 저장
- 재생 부스에서:
  - 상단 전광판에 메시지 흐름 (marquee)
  - TTS로 사연 읽기
  - 유튜브 프레임 하단 배치 — 곡 검색·재생·자동화
- 사연 5명 / 라디오 시간 10분 → **인당 2분** 자동 분배
- 다음 사연 전 자동 페이드아웃
- 종료 시 마무리 멘트 (명언·동시·속담·안전문구 **300개 저장**)

## 2. 문제점 분석 및 현실적 조정

### ⚠️ 구현 가능하지만 문제 많은 구간

| 이슈 | 원인 | 결론 |
|---|---|---|
| TTS ↔ 유튜브 타이밍 연동 | `onend` 이벤트 불안정, 사연 길이 가변 | TTS 끝난 뒤 노래 시작, 타이머는 **노래만** 카운트 |
| 유튜브 자동 검색 | IFrame API 검색 불가, YouTube Data API 키 필요 | 사연 입력 시 **유튜브 URL 직접 입력** |
| 정확한 인당 2분 보장 | TTS 시간 가변 | 총시간 ÷ 사연수 = **노래 재생 시간만** (±오차 허용) |

### ❌ 구현 불가 구간

- **유튜브 곡 자동 검색** — iframe CSP 차단, Data API 없이는 불가
- **모바일 자동재생** — iOS/Android 브라우저 정책상 차단 (PC 전용으로 설계)

### ✅ 최종 확정 스펙 변경사항

| 원안 | 확정안 |
|---|---|
| 곡제목+가수로 자동 유튜브 검색 | 사연 입력 시 유튜브 URL 직접 입력 |
| 정확히 인당 N분 | TTS 제외, 노래만 타이머 (총시간 ÷ 사연수) |
| 완전 자동 흐름 | 시작 버튼 1번 → 이후 자동 |
| 모바일 지원 | **PC 전용** 부스, 사연 입력은 모바일/태블릿 |

## 3. 추가 확정 사항

대화 중 다운쌤이 추가 확인한 내용:

- **곡 소개 TTS 포함** — "○○님의 신청곡은 [가수]의 [곡제목]입니다. 사연을 들어볼게요."
- **타이머 = 총시간 ÷ 사연수**, TTS 시간은 제외, 노래 재생 시간만 카운트
- **Firebase Firestore** 로 모바일(사연입력) ↔ PC(부스) 데이터 연동
- **GitHub Pages** 로 배포 (3개 파일: index / submission / booth)
- Firebase 프로젝트 없음 → **신규 생성 안내 포함**

## 5. 파일 목록 및 역할

### `index.html` — 설정 페이지

- Firebase apiKey, authDomain, projectId 등 6개 항목 입력
- 총 방송 시간(분), 세션 이름 입력
- 입력값 → `localStorage` 저장
- 사연 페이지 / 부스 페이지 바로가기 링크 제공

### `submission.html` — 사연 입력 (모바일/태블릿)

입력 필드:
- 이름
- 사연 내용 (최대 300자, 글자수 카운터)
- 곡 제목 / 가수 이름
- 유튜브 URL

동작:
- 제출 시 Firestore `stories` 컬렉션에 저장
- 자동 순번(`order`) 부여 (기존 최대값 + 1)
- 제출 완료 화면 표시 (이름, 신청곡, 순번 확인)

### `booth.html` — 방송 부스 (PC)

구성 요소:
- **전광판** (상단, marquee 애니메이션)
- **현재 사연 카드** (이름, 곡 정보, 사연 내용)
- **TTS 상태 표시기** (파형 애니메이션)
- **노래 타이머 바** (진행률 + 남은 시간)
- **YouTube IFrame Player** (우측)
- **사연 큐 목록** (순서, 현재/완료 상태 표시)
- **컨트롤 바** (시작 / 다음 사연 / 종료)

## 6. 방송 자동 진행 흐름

```
방송 시작 버튼
    ↓
인트로 TTS ("학급라디오 시작합니다!")
    ↓
[사연 N 시작]
    ├─ 전광판: "○○님의 사연 | 가수 — 곡제목" 흐름
    ├─ 유튜브 URL 미리 로드
    ├─ TTS: "○○님의 신청곡은 [가수]의 [곡제목]입니다. [사연내용]"
    ├─ TTS 종료 → 유튜브 재생 시작
    ├─ 타이머: (총시간 ÷ 사연수) ms 카운트다운
    └─ 타이머 종료 → 볼륨 페이드아웃 → 다음 사연
    ↓
[마지막 사연 완료]
    ├─ 종료 멘트 300개 중 랜덤 선택
    ├─ TTS: "[명언/속담/동시] 오늘 방송을 들어주셔서 감사합니다. 다음에 또 만나요!"
    └─ 종료 오버레이 표시
```

## 8. GitHub Pages 배포 가이드

**준비물:** GitHub 계정, Firebase 프로젝트

### Step 1 — Firebase 프로젝트 생성
1. [firebase.google.com](https://firebase.google.com) → 시작하기 → Google 로그인
2. 프로젝트 추가 → 이름: `classroom-radio` → Analytics 사용 안함
3. `</>` 웹 앱 추가 → 앱 이름: `radio` → **firebaseConfig 복사**
4. Firestore Database → 데이터베이스 만들기 → **테스트 모드** → 위치: `asia-northeast3 (서울)`

### Step 2 — GitHub 저장소 생성
1. github.com → New repository
2. 이름: `classroom-radio` → **Public** → Create

### Step 3 — 파일 업로드
1. Add file → Upload files
2. `index.html`, `submission.html`, `booth.html` 드래그 앤 드롭
3. Commit changes

### Step 4 — Pages 활성화
1. Settings → Pages → Branch: `main` → Save
2. 배포 주소: `https://[GitHub아이디].github.io/classroom-radio/`

### Step 5 — 초기 설정
1. `index.html` 접속 → Firebase config 붙여넣기 → 저장
2. 방송 시간, 세션 이름 입력 → 저장

### 사용 시 순서
```
① 선생님 index.html 설정 (최초 1회)
② submission.html 링크 → 학생 기기에 공유 → 사연 수집
③ 사연 다 모이면 booth.html 새로고침 → 방송 시작
```

## 9. 알려진 제한사항 / 유의점

- **booth.html은 방송 시작 전 새로고침** 필요 (실시간 리스너 아닌 스냅샷 방식)
- **PC Chrome 권장** — Web Speech API TTS, YouTube 자동재생 안정성
- Firestore 테스트 모드는 **30일 후 만료** → 이후 보안 규칙 설정 필요
- 모바일에서 booth.html 사용 시 자동재생 차단될 수 있음
- TTS 음성은 기기에 설치된 한국어 음성 엔진 품질에 따라 다름
